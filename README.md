### 该文档记录原生JS实现板球过程
1. 首先第一点，这是一个实时的游戏，需要以每秒多少帧的设定匀速渲染，framesPerSecond项可以设定帧率；
2. 每一帧需要做的事情是这样的，需要覆盖canvas画布，绘制球网，绘制球拍，绘制球，绘制分数，也可能需要绘制结束画面，who wins for now；
3. 需要记录的状态，球拍的位置，球的位置，以及分数；
3.1 首先player的球拍Y位置根据mousemove事件的clientY来定，但是需要考虑canvas画布相对于body原点的diff距离以及rootElement的scrollY偏移；
3.2 其次是球的位置，这个可以通过设置固定的x方向和y方向的球移动方向（正反）速度来更新；
#### 需要处理的特殊情况：
1. 球触碰到上下边界出现反射，这时候只需要检测球的Y数值，出现该情况时将Y方向的速度置反即可；
2. 球触碰左右边界时在球拍的范围之内，检测球的X数值，判断是否在球拍的范围内，如果在则将x方向速度置反，并且求出deltaY，即球的位置与球拍中心的diff距离，根据这个距离来确定反向速度，越大越大；如果不在范围内则可以直接得分；
3. 每次重新开始，球应该在画布中央，并且得分之后应该将球的初始方向置反，有利于输家；
4. 无敌机器人击球，时刻检测球的位置与球拍Y之间的距离，如果绝对值大于阈值就将球拍移动一定距离，可以肯定这是完全可以实现无敌机器人的！
